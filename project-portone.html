PortOne 기반 결제 및 구독 서비스

PortOne API Pessimistic Lock Transaction

프로젝트 개요

기간: 2026.03 ~ 2026.03

역할: 결제 시스템 백엔드

외부 결제 연동(PortOne API)을 통해 실제 결제 프로세스가 어떻게 동작하는지 파악하고, 동시에 다수의 사용자가 결제 요청을 보낼 때 발생하는 재고 동시성 문제를 해결한 프로젝트입니다.

주요 구현 및 기술적 의사결정

  • 동시성 제어: 비관적 락(Pessimistic Lock)을 적용하여 동시 다발적인 결제 요청 시 재고의 정합성을 완벽하게 보장했습니다.
  • 환불 보상 트랜잭션: 결제 실패 혹은 부분 환불 시 안전하게 롤백될 수 있도록 보상 트랜잭션 로직을 구현했습니다.

성과 및 결과

테스트 환경에서 결제 중단 및 데이터 꼬임 없이 정합성을 100% 유지하며 외부 API와의 안정적인 연동을 달성했습니다.
