1. 주제 선정 배경

5인 팀으로 진행한 결제 및 구독 서비스 백엔드 프로젝트입니다. PortOne API를 활용하여 실제 결제 시스템의 흐름을 파악하고, 결제 과정에서 발생할 수 있는 데이터 불일치 및 동시성 문제를 예방하며 결제 위변조, 중복 처리, 재고 불일치 문제를 사전에 방지할 수 있는 안정적인 결제 시스템 구현을 목표로 했습니다.

2. 담당 역할

  • 결제 검증, 재고 차감, 환불 처리 기능 백엔드 API 설계 및 구현 담당
  • PortOne API 연동 기반 결제 재검증 및 웹훅 멱등성 처리 구조 설계
  • 비관적 락(Pessimistic Lock) 기반 재고 정합성 완벽 보장
  • 실패 시 안전하게 롤백을 보장하는 환불 보상 트랜잭션 설계

3. 기술 스택

Java / Spring Boot MySQL / JPA PortOne 결제 API Pessimistic Lock Compensating Transaction

5. 주요 기능

  • 안전한 결제 검증: 클라이언트 측 데이터 조작 방지를 위한 결제 금액 단건 조회 및 자동 취소
  • 동시 다발 결제 처리: 동시 요청 시에도 정합성을 잃지 않는 재고 차감 및 복구 시스템
  • 환불 및 정합성 보장: 부분 환불 및 전액 환불 시 재고/포인트 자동 복구 및 이력 관리

6. 기술적 의사결정 및 트러블슈팅

PortOne 단건 조회 재검증 및 웹훅 멱등 처리로 결제 위변조 방지

배경

클라이언트 결제 완료 요청과 웹훅 이벤트를 단순히 신뢰할 경우 결제 금액 위변조 및 동일 결제 건의 중복 처리 위험이 존재했습니다. 또한 동일 imp_uid에 대한 웹훅이 여러 번 수신될 경우 중복 결제가 승인될 가능성이 있었습니다.

기술적 의사결정

결제 데이터의 신뢰성을 절대적으로 확보하기 위해, 클라이언트 요청 값을 그대로 사용하지 않고 PortOne 단건 조회 API를 통한 서버 측 재검증 방식을 최우선으로 채택했습니다. 또한 웹훅 재전송 환경을 고려하여 멱등성(Idempotency) 기반 처리 구조를 설계해 어떠한 경우에도 중복 적용되지 않도록 조치했습니다.

주요 구현

  • PortOne 단건 조회 API 기반 결제 금액 재검증
  • 주문 금액과 결제 금액 불일치 시 자동 결제 취소 처리 구현
  • imp_uid 기반 중복 결제 방지 로직 적용
  • 결제 상태 단방향 전이(READY → PAID → CANCELLED) 엄격한 설계

성과

결제 금액 위변조 차단율 100%를 달성하였으며, 웹훅 중복 수신으로 인한 중복 처리 건수 0건을 유지했습니다. 결과적으로 사용자가 결제를 안전하게 진행할 수 있는 견고한 검증 체계를 확보했습니다.

결제 확정 시 비관적 락 적용을 통한 재고 정합성 확보

배경

동일 상품에 대한 다수의 동시 결제가 발생할 경우, 재고 조회와 차감 사이의 Race Condition으로 인해 초과 판매 및 음수 재고 현상이 일어날 수 있는 심각한 위험이 있었습니다.

기술적 의사결정

결제 트래픽 특성상 동시 충돌 가능성이 매우 높다고 판단하여, 재시도 비용이 발생하는 낙관적 락(Optimistic Lock) 대신 DB 레벨에서 즉시 배타적 제어가 가능한 비관적 락(Pessimistic Lock)을 적용했습니다.

주요 구현

  • Product 테이블 조회 시 PESSIMISTIC_WRITE 락 적용
  • 재고 차감 로직과 결제 확정 로직을 단일 트랜잭션으로 강력하게 결합 처리
  • 재고 부족 시 PortOne 결제 롤백(취소) 자동 연동 구현
  • 결제 실패 시 차감되었던 재고 및 사용된 포인트의 안전한 복구 로직 포함

성과

강도 높은 동시 결제 테스트 환경에서도 초과 판매 0건을 달성하며 재고 데이터 정합성을 완벽히 확보했습니다. 결제 실패 시의 데이터 불일치 가능성을 원천적으로 제거하여 신뢰도 높은 구매 환경을 구축했습니다.

환불 보상 트랜잭션 적용을 통한 결제 정합성 확보

배경

환불 요청 시 'PG 취소 - 포인트 회수 - 포인트 복구 - 재고 복구'의 4단계가 순차적으로 수행되어야 합니다. 이 중 일부 단계에서 예외가 발생할 경우, 결제 상태와 실제 데이터(재고/포인트) 간의 치명적인 불일치가 발생할 수 있었습니다.

기술적 의사결정

전체 환불 프로세스를 하나의 비즈니스 트랜잭션 단위로 묶고, 중간 단계 실패 시 안전하게 상태를 되돌릴 수 있는 보상 트랜잭션(Compensating Transaction) 구조를 설계하여 정합성 위반 요소를 사전 차단했습니다.

주요 구현

  • 환불 요청 인입 시 기존 결제 상태 사전 검증 로직 구현
  • PortOne 외부 API 결제 취소 안정적 연동 및 예외 처리
  • 결제로 적립된 포인트 자동 회수 및 사용했던 포인트 복구 처리
  • 상품 재고의 정확한 복구 및 환불 이력 데이터 생성 구현

성과

실운영 시나리오 테스트 결과, 환불 시 PG·포인트·재고 데이터 불일치가 단 1건도 발생하지 않았습니다. 번거로운 수동 데이터 보정 작업 필요성을 완전히 제거하여 운영 효율을 크게 높였습니다.
